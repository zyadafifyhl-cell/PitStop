Car Wash Multi-Branch Roles & RLS Supabase Auth

1. Vision

PitStop lets a shop owner run one brand with many branches. The owner keeps full control; each branch has a branch manager with a separate login; employees are roster records only (no app accounts).

2. Role Hierarchy

Owner (shop brand account)
  ├── Creates & manages branches
  ├── Creates / links Branch Manager accounts (email + password)
  └── Sees all branches + global reports

Branch Manager (one branch)
  ├── Manages daily operations for assigned branch only
  ├── Adds / edits employees (info only, no login)
  └── Handles bookings, schedule, services for that branch

Employees
  └── Name, phone, job title, notes — stored in branch_employees (no login)

6. Employee Records (No Login)

Employees are not Supabase Auth users. Branch manager (or owner) adds:

  • Full name (required)
  • Phone (optional)
  • Job title — e.g. washer, cashier (optional)
  • Notes — shift, skills (optional)

Stored in branch_employees. Used for internal roster — future: assign to bookings / shifts.

7. Owner Creates Branch Manager — Workflow

Owner logs in (wash@demo.com) | v Opens Branch → Branch Managers section | +--> Link existing email (user already registered) | | | v | RPC assign_branch_manager_by_email | +--> Create new login (email + password + name) | v assign_branch_manager (role before signUp) | v signUp → restore owner session | v Branch manager logs in → sees own branch only

8. Booking Status Flow (Owner / Manager)

pending → confirmed (accept) / cancelled (reject + note) confirmed → in_progress → done confirmed → no_show cancelled ← customer or owner/manager
